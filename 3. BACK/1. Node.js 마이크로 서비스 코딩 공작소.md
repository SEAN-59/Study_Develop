## Chapter 01. 마이크로 서비스의 개요
### 1. 모놀리식 아키텍쳐
- 전통적인 아키텍쳐로 하나의 어플리케이션 안에 모든 컴포넌트를 포함하는 구조
    - 구조가 단순해 개발과 배포가 간편하다는 장점
### 2. 모놀리식 아키텍쳐의 한계
- 개발과 배포가 쉽다는 장점은 있지만 일체형이라 한계가 존재
    - 분산 처리에 비효율
    - 코드 관리 어려움
    - 신기술 적용 힘듬
### 3. 분산 아키텍쳐의 필요성
- 한계 극복위해 마이크로 서비스 아키텍쳐의 등장
    - 코드 구조가 독립적
    - 기능별 분산 구조
    - 기능별 신기술 적용 가능
